프로그래밍 성능은 어떻게 측정할까?

프로그램의 성능을 측정하려면, 데이터의 입력크기, 하드웨어 성능, 운영체제 성능 등등 많은 것들을 고려해야한다. 여러가지 다양한 상황과 조건들이 있기 때문에 실질적으로 성능을 정확하게 측정하는 것은 불가능하다.

따라서 프로그래밍 성능을 측정하는 상대적인 표기법을 정했는데, 그게 바로 `Big-O notation`이다.

이 `Big-O`표기법으로 시간복잡도를 나타내어 효율적인 알고리즘을 구현할 수 있다.

## 👋 잠깐! 로그(log)란?

숫자 2와 8이 있을때 2를 `몇번` 곱해야 8이 나올까? 그 `몇번`을 구하는 것이 log이다.

위 예시를 로그로 표현하면 아래와 같다.

> log<sub>2</sub>8 = 3

## 🛠 시간복잡도 코드로 보기

### 👷🏾‍♀️ O(n) - 선형시간

`O(n)`은 입력받은 `n`의 크기만큼 루프를 도는 단순한 선형시간이다.

```javascript
for (let i = 0; i < n; i++) {
  //...
}
```

### 👷🏾‍♀️ O(log n) - 지수시간

아래 코드에서는 입력받은 `n`의 2배씩 결과값을 출력하고 있으므로, 밑을 2로 가진 log 값을 가지게 된다.
얼핏 보면 선형시간과 비슷해 보이지만 어마마한 차이가 있다. 만약 n이 1024일 경우 선형시간에서는 1024번을 돌지만 O(log n)같은 경우에는 10번만 돌기 때문에 거의 100배 넘는 시간이 차이가 나게 된다.

```javascript
for (let i = 1; i <= n; i *= 2) {
  //...
}
```

### 👷🏾‍♀️ O(n log n) - 선형지수시간

선형 지수 시간은 `선형시간 * 지수시간` 이다.

```javascript
for(let i = 0; i < n; i +=1) {
	for(let j =1; j<=; j *=2) {
		//...
	}
}
```

### 👷🏾‍♀️ O(n<sup>2</sup>) - 이차시간

이차시간은 n의 제곱 만큼, 다시말해서 선형시간의 두배만큼의 루프를 돌게 된다.

```javascript
for (let i = 0; i < n; i++) {
  for (let i = 0; i < n; i++) {
    //...
  }
}
```

이외에 O(2<sup>n</sup>)-지수시간과 O(n!)-팩토리얼 시간은 가급적 사용하면 안되는 경우라 샘플이 없다.

### Big-O의 점근상한

Big-O에서 점근상한은 만약 시간복잡도가 O(log n)일때 최대 소요 시간이 O(log n)을 넘지 않는 다는 의미이다. 표기는 아래와 같이 `f(n) = O(g(n))`으로 한다.

### 🧭 Big-O의 법칙

#### 1. 계수법칙

계수법칙은 입력크기에 얼마나 큰 상수를 곱하여도 동일한 시간 복잡도를 가진다는 것이다.
따라서 아래 두 반복문에서 두번째 반복문이 첫번째 반복문보다 입력크기가 5배가 넘지만
결국엔 같은 선형시간으로, kf(n) = f(n)이 된다.

```javascript
for (let i = 0; i < n; i++) {
  //...
}

for (let i = 0; i < n * 5; i++) {
  //...
}
```

#### 2. 합의 법칙

빅오끼리는 서로 더할수 있다. 아래 두 루프를 합쳐서 O(n + m)으로 표기 할 수 있다.
계수 법칙에 의해 5는 의미가 없으므로 사라지게 된다.

```javascript
for (let i = 0; i < n; i += 1) {
  //..
}

for (let i = 0; i < m * 5; i += 1) {
  //...
}
```

#### 3. 곱의 법칙

빅오끼리는 서로 곱할 수 있다. 아래 두 루프를 곱해서 O(n<sup>2</sup>)로 표기할 수 있다. 마찬가지로 계수법칙에 의해 5는 사라진다.

```javascript
for (let i = 0; i < n; i += 1) {
  for (let j = 0; j < n * 5; j += 1) {
    // ...
  }
}
```

#### 4. 다항 법칙

f(n)이 k차 다항식이면 f(n)은 O(n<sup></sup>)

```javascript
for (let i = 0; i < n * n * n; i += 1) {
  //...
}
```

⭐️ 여기서 기억하면 될 것은 아래 두가지이다.

1. 계수법칙에서 상수항은 무시
2. 가장 작은 항은 무시

```javascript
//O(n^2 + n)이지만 작은항을 무시하고 O(n^2)로 표기할 수 있다.

for (let i = 0; i < n; i += 1) {
  //..
}

for (let i = 0; i < n; i += 1) {
  for (let i = 0; i < n; i += 1) {
    //..
  }
}
```

### 👍 Javascript에서 성능을 측정하는 방법!(꿀팁) - Date 객체를 이용

Date객체를 이용하면 오래된 브라우저에서도 사용 할 수 있기 때문에 가장 안전하다.

```javascript
const start = new Date().getTime()

// 로직작성

const end = new Date().getTime()

console.log(end - start) // 로직이 끝난시간에서 시작한 시간을 뺴준다.
```
