프로그램이 실행되기 위해서는 프로그램이 메모리에 로드 되어야 한다.
이 메모리에 프로그램에 필요한 변수들, 메소드 등이 저장되는데, 이러한 메모리 공간은 운영체제의 메모리 관리자에 의해 제공된다.

프로그램이 운영체제로부터 할당받는 대표적인 메모리 공간에는 네가지가 있다.

## 1. ✅ 코드 영역

코드 영역은 실행할 프로그램의 코드가 저장되는 영역으로 텍스트 영역이라고도 부른다. CPU는 코드 영역에 저장된 명령어를 하나씩 가져가서 처리하게 된다.

## 2. ✅ 데이터 영역

메모리의 데이터 영역은 프로그램의 `전역 변수`와 `정적 변수`가 저장되는 영역이다. 데이터 영역은 `프로그램의 시작`과 함께 할당되며, 프로그램이 종료되면 소멸한다.

## 3. ✅ 스택 영역

스택 영역은 함수의 호출과 관계되는 `지역변수`와 `매개변수`가 저장되는 영역이다. 스택 영역은 함수의 호출과 함께 할당되며, 함수의 호출이 완료 되면 소멸하게 된다. 이렇게 스택 영역에 저장되는 함수의 호출 정보를 스택 프레임이라고 한다.

스택은 LIFO방식에 따라 동작하기 때문에, 가장 늦게 저장된 데이터가 가장 먼저 인출되고 높은 주소에서 낮은 주소의 방향으로 할당된다.

## 4. ✅ 힙 영역

힙 영역은 사용자가 직접 관리할 수 있는, `꼭 그래야만 하는` 메모리 영역이다. 힙 영역은 사용자에 의해 메모리 공간이 동적으로 할당되고 해제된다. 따라서 메모리의 낮은 주소에서 높은 주소의 방향으로 할당된다.
