## 데이터

데이터는 하나 이상의 사람이나 사물에 관한 `정성적 또는 정량적 값`의 집합이다. 정의는 이렇게 되어있지만 쉽게 말하면, 기록해 놓고 필요할때마다 찾아서 보는 자료를 데이터라고 한다.

## 데이터베이스의 등장 이전

데이터베이스가 등장하기 전에는 데이터를 처리할 때 파일 처리 방식에 의존하였다. 파일 처리 방식은 1차원적인 저장 시스템으로, 각 사용자 혹은 응용 프로그램은 필요한 파일을 독립적으로 소유하고 관리한다. 그렇다보니 각 사용자들이 중복된 데이터를 가지고 있을 수도 있고, 각 사용자들이 독립적으로 관리하기 때문에 데이터가 종속성을 가지게 된다.

## 데이터 종속성의 문제

여러 사용자가 각각의 데이터를 관리하게 되면서, 사용자와 데이터는 1:1의 관계를 가진다. 따라서 데이터가 보조기억장치에 저장되는 구성방식이나 접근방식에 맞게 응용 프로그램이 구성되어야 한다. 그래서 만약 파일의 구성요소, 접근방식 등이 변경되면 응용 프로그램도 함께 변경되어야 하는 문제점이 생긴다.

## 데이터 중복성의 문제

데이터 종속성의 문제와 동일한 원인으로, 각각의 사용자 혹은 프로그램이 데이터를 관리하기 때문에 자연스럽게 각각의 사용자는 동일한 데이터를 가지고 있을 가능성이 있다. 그렇게 되면 해당 데이터는 일관성을 유지하기가 쉽지 않고, 보안에 있어서도 취약한 문제점이 있다.

## 데이터베이스

위와 같은 문제점들을 해결하기 위해서 나타난 것이 바로 데이터베이스이다.
데이터베이스는 여러 사람이 사용할 수 있도록 통합해서 관리하는 데이터의 집합을 말한다. 데이터베이스가 등장하면서, 기존의 파일처리 시스템과 달리 하나의 데이터를 여러 사용자 혹은 프로그램들이 공유해서 사용하기 때문에, 독립적으로 데이터를 사용함으로서 기존의 종속성이나 중복성의 문제점들을 해결 해줄 수 있게 되었다.

그리고 이 데이터베이스를 실제로 구현한 것이 `데이터베이스 관리 시스템(DBMS)`인데, 데이터베이스를 관리하는 시스템, 소프트웨어나 프로그램이다.

## RDBMS(관계형 DBMS)

초창기의 DBMS는 탐색형 DBMS였는데, 데이터를 조회하고 탐색하는 성능이 별로 좋지 못했다. 이런 단점을 보완하고자 등장한 것이 RDMS다. RDBMS(Relationship Database Management System)는 가장 많이 사용되는 DBMS로 SQL을 기반으로 하고 있다.

## RDBMS 특징

- 중복 데이터 저장을 최소화 한다. (최소화 한다는 것이지 아예 못한다는 것은 아니다)
- SQL로 데이터를 손쉽게 처리할 수 있다.
- 트랜잭션 처리를 할 수 있다.

> 트랜잭션: RDBMS는 트랜잭션 처리를 통해, 데이터 가공작업에 있어 성공되면 작업 결과에 적용하고 오류가 발생하면 작업 자체를 취소한다.

- 데이터 무결성을 보장한다.

> 무결성: 데이터의 정확성을 보장한다. 정확하고 올바른 데이터가 저장되도록 보장하는 것을 의미한다.

## RDBMS 종류

가장 많이 사용되는 대표적인 3가지 RDBMS는 아래와 같다.

- Oracle
- MySQL
- SQL Server

오라클은 제품 이름이자, 해당 RDBMS를 만든 회사 이름이다. 역사도 오래되고 성능도 좋지만, 다른 RDBMS보다 가격이 비싸다는 단점이 있다고 한다. 그리고 두번째로 많이 사용하고 있는 MySQL이 있고, 요즘에는 PostgresSQL도 점점 사용하는 곳이 많아지는 듯 하다.

## MySQL

MySQL은 오픈소스라서 무료로 사용할 수 있고, 성능이 나쁘지 않아서 큰 인기를 끌고 있다. 초반에는 성능이 좋지 못했지만, 꾸준히 기능 개선을 해와서 다른 제품과 비교해도 성능이 떨어지지 않는다.

원래는 무료였지만, 오라클로 넘어간 뒤로는 무료 버전과 유료 버전이 따로 나오게 되었다

MySQL을 설치하면 기본적으로 Root라는 사용자 계정이 만들어지는데 이 계정이 관리자 계정이 된다. 그리고 root 계정말고도 새로운ㅇ 사용자 계정을 만들 수 도 있다.

## MariaDB

MariaDB는 MySQL의 자매라고 한다. MariaDB는 오라클이 MySQL을 인수하면서 MySQL의 오픈 소스 사상을 이어가기 위해 MariaDB를 만들었다. 태생이 같기 때문에 MySQL에서 사용하는 방식을 MariaDB에도 그대로 적용할 수 있다고 한다.

## MySQL 서버

MySQL 서버는 MySQL이라는 RDBMS를 말하는데, 이름에 서버가 붙은 이유는 DBMS 자체가 서버 역할을 하기 떄문이다. 따라서 일반적으로 MySQL이라고 하면 MySQL 서버를 의미한다.

PC에 MySQL을 설치하면 해당 PC가 서버가 되고 PC에서 다른 도구로 MySQL을 접속하면 그 PC가 클라이언트가 된다고 한다. 따라서 MySQL을 설치하게 되면 내 컴퓨터가 서버인 동시에 클라이언트가 되는 것이다.

## MySQL의 데이터베이스와 스키마

데이터베이스는 데이터들의 집합체라고 한다. 그런데 MySQL에서의 데이터베이스는 테이블을 포함해 각종 데이터베이스 객체(데이터를 저장하거나 데이터와 관련 있는 구조체)를 담아 놓는 논리적인 저장 공간이 된다. 일반적으로 데이터베이스와 스키마는 다른 RDBMS에서는 다른 개념처럼 사용되지만 MySQL에서는 데이터베이스와 스키마가 같은 개념으로 활용된다.
