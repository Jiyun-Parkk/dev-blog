요즘 프로젝트를 하면서 느끼는건 생각보다 git이 너무 헷갈리고 어렵다는점...이다.

그 중에서도 협업을 할때 현재 작업중일때 다른 작업자가 merge 한 내용을 `pull` 받으면 현재 나의 로컬 작업상태와
pull 받으려는 branch의 상태가 달라서 `conflict`날때가 있다.

그때마다 바보같이.. 작업내용을 따로 백업해놓고 받고..하는 무식한 짓을 했는데 오늘 방법을 알았다🥲

## 상황

- 다른 사람이 작업한 코드가 반영된 브랜치 내용을 pull
- 현재 나도 동일한 브랜치에서 작업중

## 1. stash로 임시저장공간에 내 작업물 저장하기

아래 명령어를 작성하면 임시저장공간에 내 작업물이 저장된다.

```bash
git stash
```

## 2. git fetch => git pull origin [branch]로 저장소 연결 & 파일 내려받기

> git fetch: 원격저장소의 commit들을 다운, 원격 브랜치가 가르키는 곳을 업데이트

`git fetch`를 하면 현재 올라와 있는 git과 연결시켜준다. 그리고 pull을 받아서 최신화된 파일을 내려받는다.

## 3. 로컬 저장소 최신화 하기

내가 작업한 코드는 아직 로컬에 남아있기 때문에, rebase(덮어씌우기)를 해줘야 한다.
rebase는 브랜치끼리의 작업을 접목하는 방법이다. 리베이스는 기본적으로 커밋들을 모아서 복사한뒤,
다른 곳에 떨궈 놓는 것이다.

## 4. 임시저장한 내 작업파일 다시 가져오기

```bash

git stash pop  #임시저장소에서 내 코드를 다시 꺼낸다
git add 파일
git commit -m "메세지"
git push origin branch
```
