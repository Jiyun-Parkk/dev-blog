아는 지인들과 팀을 만들어서 사이드 프로젝트를 시작했는데, 드디어 우여곡절 끝에 개발작업이 시작되었다.(길고 길었던 기획과정..ㅎㅎ)
가난한 취준생들이 처음 팀프로젝트를 하려면 어떻게 해야하나 이것저것 찾아보던 중 `git organization`을 알게 되었다.

## 1. git organization 생성하기

git에 있는 organization을 활용하면 해당 프로젝트의 레포만 확인 할 수 있는 고유 git이 생성되어
작업하기도 편하고, 나중에 다른 사람에게 공유하기에도 좋다. organization은 아래 이미지 처럼 `Create organization`으로 만들면 된다.
![image](https://user-images.githubusercontent.com/72537762/208643782-983a43e7-7742-45a1-8b0c-730cf9929980.png)

organization을 생성했으면 이제 프로젝트 레포지토리를 만들고, 프로필도 예쁘게 꾸며준다 ㅎㅎ

## 2. organization people 초대

orgnization에 people 탭에서 같이 프로젝트를 하는 팀원들을 초대하고, 팀원들에게 각자 people탭에서
priviate으로 되어 있는 설정으로 public으로 변경하라고 요청한다.

이 설정은 본인 밖에 변경이 안된다.

![image](https://user-images.githubusercontent.com/72537762/208645884-cf4f0c4c-e63c-4fef-9ca1-47fcc959521a.png)

이 설정을 public으로 하게 되면 내 깃 계정 화면에 예쁜 organization 뱃지가 하나 추가된다ㅎㅎ

내 깃 페이지가 한층 더 업그레이드 된 듯한 느낌..!
![image](https://user-images.githubusercontent.com/72537762/208646276-73e55316-ebb0-40cf-8557-8b04d7d85a80.png)

## 3. project 생성하기

git의 project탭에서 프로젝트를 생성하면, 진행할 프로젝트의 작업 기록들을 한번에 볼 수 있다.
template이 여러가지가 있는데, 나는 간단하게 표로 보는게 편해서 표 형식으로 만들었고 각각 commit과 merge가 일어날때
어떤 리뷰어가 있었는지 등등의 기록을 대략적으로 확인 할 수 있다.

마지막으로 레포지토리에 project 소스를 push 하고 작업을 이어 가면 된다.
