## Overloading

변수가 같은 이름을 가질 수 없듯이, 메서드도 각각 다른 이름을 사용해야 한다. 하지만 자바에서는 한 클래스 내에 동일한 이름의 메서드가 있더라도, 매개변수의 개수 또는 타입이 다르면 같은 이름을 사용해서 메서드를 정의할 수 있다.

이와 같이 한 클래스 내에서 같은 이름의 메서드를 여러개 정의하는 것을 `오버로딩`이라 한다. 오버로딩이 성립하기 위해서는 다음 조건을 만족해야 한다.

1. 메서드 이름이 같아야 한다.
2. 매개변수의 개수 또는 타입이 달라야 한다.
3. 반환 타입은 관계 없다.

만약 위의 조건을 만족시키지 못하는 메서드는 중복정의로 간주되어 컴파일시에 에러가 발생한다. 오버로딩된 메서드들은 매개변수에 의해서만 구별될 수 있으므로 반환 타입은 오버로딩을 구현하는데 아무런 영향을 주지 못한다.

이런 오버로딩은 단순히 내가 만든 클래스에서의 메서드로만 생각했는데, 의외로 자주 사용하는 `println` 메서드가 오버로딩 되고 있는 대표적인 예라고 한다.

println메서드를 호출할때 매개변수로 넘겨주는 값의 타입에 따라서 `PrintStream`클래스에 정의된 10개의 오버로딩 된 `println`메서드를 정의해놓고 있다.

위 조건들을 토대로 오버로딩이 성립되는 케이스를 보면 아래와 같다.

```java
int test(int a, int b) {
	return a + b;
}

double test(double x, double y) {
	return x + y;
}
```

위 예시코드에서는 반환 타입과 관계 없이, 매개변수의 타입이 다르므로 오버로딩이 성립된다.

## Overriding

오버라이딩은 상속관계에 있는 부모 클래스에서 이미 정의한 메소드를 자식 클래스에서 같은 이름으로 다시 정의하는 것이다. 자바에서 자식 클래스는 부모 클래스의 private 멤버를 제외한 모든 메서드를 상속 받는데, 만약 부모에게 상속받은 메서드에 추가적으로 수정하여 사용하고 싶다면 같은 이름으로 재정의하여 사용 할 수 있다는 것이다.

오버라이딩의 조건은 아래와 같다.

1. 메서드의 동작만을 재정의 하는 것이므로, 메서드의 선언부는 기존 메서드와 완전히 같아야 한다. 하지만 메서드의 반환 타입은 부모 클래스의 반환 타입으로 타입 변환 할 수 있는 타입이라면 변경할 수 있다.

2. 부모 클래스의 메서드보다 접근 제어자를 더 좁은 범위로 변경할 수 없다.

3. 부모 클래스의 메서드보다 더 큰 범위의 예외를 선언할 수 없다.

오버라이딩의 예시코드는 아래와 같다. 아래와 같이 상속받은 부모의 메서드와 동일한 이름으로 구현부만 수정했는데, 결과적으로는 오버라이딩된 구현부가 출력된다.

```java

class Parent {
  void display() {
    System.out.println("a");
  }

}

class Child extends Parent {
  void display() {
    System.out.println("b");
  }
}

Child child = new Child();
child.display(); // b
```
