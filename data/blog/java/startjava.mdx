## 자바 시작하기

프로그래밍이란 컴퓨터에게 일을 시키는 것이다. 컴퓨터는 0,1로 이루어진 언어로 소통하는데 이것을 바이너리 언어라고 한다. 흔히 개발자들이 사용하는 언어들은 사람이 이해하는 언어이고, 컴퓨터가 이해할 수 있는 언어가 아니다. 따라서 컴퓨터가 이해하는 언어가 되기 위해서는 통역과정이 필요한 것이다.

## JVM

위와 같은 이유로 자바 또한 실행되기 위해서는 통역해주는 기계가 하나 필요하다. 각 운영체제에 맞는 JDK를 설치하면 자바 프로그램을 실행할 수 있는 JVM이 함께 설치된다. JVM에는 두가지 기능이 있다.

> JDK = JRE(자바 런타임 환경 JVM + library classes) + Development Tools

1. 어떤 운영체제 상에서도 실행 될 수 있게 만들어준다
2. 자바 프로그램의 메모리를 효율적으로 관리 & 최적화 해준다

> Garbage Collection: JVM이 메모리를 관리하는 프로세스 프로그램 상에서 사용하지 않는 메모리를 지속적으로 찾아 제거함으로써, 효율적인 메모리 관리를 가능하게 한다.

JVM(Java Virtual Machine)은 자바 코드를 .class 바이트 코드로 변환해주고, 이 코드가 JVM 환경에서 실행된다. 다시말해서, 컴퓨터가 이해할 수 있는 언어로 변환시켜 주는 것이다.

## JRE

이 JVM은 JRE에 포함되어 있는데, JRE는 `자바 클래스 라이브러리`, `자바 가상 머신`, `자바 클래스 로더`를 포함하고 있다. 클래스 로더, 클래스 라이브러리를 통해 작성한 자바 코드를 라이브러리와 결합한 후 JVM에 넘겨 실행 시키게 된다. JRE는 그 자체로 특별한 기능이 있는 것 보다는 JVM이 원활하게 잘 작동할 수 있도록 환경을 맞춰주는 역할을 하게 되는 것이다.

### Java 메모리 관리

- 힙 : 자바가 변수 내용을 저장하는 장소
- 스택 : 자바가 함수 실행 및 변수 참조를 저장하는 장소
- 메타 스페이스 : 자바가 클래스 정의와 같이 프로그램에서 변화하지 않는 정보를 저장하는 장소

## 자바 프로그래밍 하기

자바에서 최소단위는 클래스다. 파일이름.java라는 파일을 생성했다면, 해당 파일에서는 `파일이름`이라는 이름을 가진 클래스가 반드시 존재해야 한다.

이것이 하나의 컴포넌트가 되어 조립식 프로그래밍이 가능하다는 것이 자바의 가장 큰 특징이자 장점이다.

자바에서 반드시 존재해야 하는 것은 두가지가 있다.

1. 파일이름과 동일한 이름의 public class
2. main() 메소드

자바는 기본적으로 `main()`메소드의 실행문들을 모두 실행하는 것으로 프로그래밍을 시작하고 끝내게 된다.

```java
//Test.java

public class Test {
	public static void main() {
		System.out.println("Hello world~");
	}
}
```
