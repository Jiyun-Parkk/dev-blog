컴포넌트는 UI를 재사용 가능한 개별적인 조각으로 나누는 것을 말한다. 여기서 중요한 것은 바로 `재사용 가능`이다. 따라서 컴포넌트를 만들때는 굳이 재사용하지 않는 것은 분리 하지 않는 것이 좋다. 컴포넌트를 만들때는 함수형 컴포넌트와 클래스형 컴포넌트 두가지가 있다.

리액트 16버전 이전에는 주로 class 컴포넌트만 사용했다. 왜냐하면 상태관리나 리액트의 생명주기 메소드를 클래스에서만 사용 할 수 있었기 때문이다.

하지만 리액트 16버전 이후에 `hook`이 추가되면서 함수형 컴포넌트에서도 클래스 컴포넌트와 동일한 기능을 구현 할 수 있게 되었고, 지금은 함수형 컴포넌트로 제작하는 것을 권장하고 있다.

먼저 컴포넌트를 만들때 주의해야 할 점을 살펴보자

## 1. 컴포넌트 이름은 대문자로 시작하자

컴포넌트를 만들 때 주의해야 할 것이 있다. 컴포넌트의 이름은 반드시 대문자로 시작해야 한다는 것이다.
그 이유는 기존의 `HTML`태그와 구분 짓기 위함이다. React에서는 JSX로 컴포넌트를 만들기때문에, 컴포넌트와 일반 HTML 태그를 함께 사용한다. 따라서 HTML 태그와 구분 지을 방법이 필요 한 것이다.

## 2. props

props를 넘겨 줄 때는 아래와 같이 넘겨줄 수 있다.

```javascript
const sample = 'hello world'

<MyComponent message='hello world'/>
<MyComponent message={'hello world'}/>
<MyComponent message={sample}/>
```

### ✅ props의 기본값은 true

Props에 어떤 값도 넘겨주지 않을 경우 기본값은 `true`가 된다. 하지만 일반적으로 props에 값을 전달하지 않는 것은 권장하지 않는데, javascript es6 문법에서 object를 단축해서 사용하는 것과 헷갈릴 수 있기 때문이다.

그럼에도 이런 동작방식이 허용되는 이유는 HTML과 동작 방식을 일치시키기 위해 남겨진 것이라 한다.

```javascript
<MyComponent autocomplete /> // autocomplete = true
// const obj = {name} // name: name
// 위 문법과 혼동될 수 있으므로 props 값을 안써주는 것은 권장 하지 않음!
```

### ✅ React의 컴포넌트는 순수함수로 동작해야한다

> 순수함수는 동일한 인자를 입력했을 때 항상 같은 값을 리턴하는 함수를 말한다.

따라서 컴포넌트에서 전달되는 props는 직접적으로 변경하면 안되고, 외부 변수에 영향을 줌으로서 부수효과를 줘도 안된다.
