퍼블리싱과 프론트개발을 구분짓는 가장 큰 것은 바로 서버와의 통신이다. 이 서버와의 통신을 하기
위해서는 보통 `API`를 가져온다고 하는데 맨날 `API`를 활용하면서, 막상 정확히 모르는 것 같아 정리해보기로 했다.

## 1. API는 무엇인가?

API는 `Application Programming Interface`의 줄임말이다.

- API의 Application : 고유한 기능을 가진 모든 소프트웨어
- API의 Interface : 두 어플리케이션 간의 서비스 계약

여기서 말하는 계약은 요청과 응답을 사용해서 두 어플리케이션이 서로 통신하는 방법을 정의한다.
API 문서에는 개발자가 이러한 요청과 응답을 구성하는 방법에 대한 정보가 들어있다.

## 2. API는 어떻게 작동하나요?

    API의 아키텍처는 클라이언트에서 요청을 보내면 서버에서 응답을 보내는 것으로 구성되어 있다.
    API는 생성된 시기와 이유에 따라 네가지 방식으로 작동 할 수 있다.

### 1) SOAP API

    단순 객체 접근 프로토콜을 사용한다. 클라이언트와 서버는 XML을 사용하여 메시지를 교환한다.
    과거에 많이 사용되었고 유연성이 떨어진다.

### 2) RPC API

    이 API는 원격 프로시저 호출이라고 한다. 클라이언트가 서버에서 함수나 프로시저를 완료하면
    서버가 출력을 클라이언트로 다시 전송한다.

> 프로시저: 데이터베이스에 대한 일련의 작업을 정리한 절차를 관계형 데이터베이스 관리 시스템에 저장한 것을 말한다.

### 3) Websocket API

    JSON 객체를 사용하여 데이터를 전달하는 최신 웹 API 개발이다. websocket API는 클라이언트 앱과 서버 간의 양방향 통신을 지원한다.
    서버가 연결된 클라이언트에 콜백 메시지를 전송 할 수 있어서 Rest API 보다 효율적이다.
    이 API를 활용하면 채팅이나 라이브 커버스 같은 실시간 통신이 가능해진다.

### 4) REST API

    웹에서 가장 많이 볼 수 있고, 가장 많이 사용되는 유연한 API이다.

- 클라이언트가 서버에 요청을 데이터로 전송
- 서버가 클라이언트 입력을 사용하여 내부 함수를 시작
- 출력 데이터를 다시 클라이언트에 반환
