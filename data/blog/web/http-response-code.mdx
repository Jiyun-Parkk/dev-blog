## 1. 200

| code | 설명                                                                            |
| ---- | ------------------------------------------------------------------------------- |
| 200  | 클라이언트의 요청을 정상적으로 수행함                                           |
| 201  | 클라이언트가 어떠한 리소스 생성을 요청, 리소스가 성공적으로 생성됨(POST 작업시) |

## 2. 400

| code | 설명                                                                                      |
| ---- | ----------------------------------------------------------------------------------------- |
| 400  | 클라이언트의 요청이 부적절 할 경우                                                        |
| 401  | 클라이언트가 인증되지 않은 상태에서 보호된 리소스를 요청 했을 경우                        |
|      | ex) 로그인 하지 않은 유져가 로그인 했을 때                                                |
| 403  | 유저 인증상태와 관계 없이 응답하고 싶지 않은 리소스를 클라이언트가 요청했을 경우          |
|      | 403 보다는 400이나 404를 사용 할 것을 권고, 403 자체는 리소스가 존재한다는 뜻             |
| 405  | 클라이언트가 요청한 리소스에서는 사용 불가능한 Method를 이용 했을 경우 사용하는 응답 코드 |

## 3. 300, 500

| code | 설명                                                     |
| ---- | -------------------------------------------------------- |
| 301  | 클라이언트가 요청한 리소스에 대한 URI가 변경 되었을 경우 |
|      | 응답 시 Location header에 변경된 URI를 적어 줘야 한다.   |
| 500  | 서버에 문제가 있을 경우                                  |
|      | 서버통신 시간지연, 트래픽 과부화, 구문 에러              |
